
* Plugin Template

For downstream developers, this serves as an example repo for creating plugins
and exposing them to be used inside a Paradise client.

Much of this is subject to change, but the core logic of a plugin should be
more or less safe. The idea being that you'd write the plugin in
a similar manner to how you might contribute directly to the main repo.
It should feel very fluid and native.

At the current moment, building and testing is more or less inline the
main repo and move out when ready to deploy as a plugin.

This will not be the mechanism for too long, but it is mentioned
to provide clarity in the current simple developer process.

Later it would be far more reasonable to be able to declare
the upstream as a dependency and launch the plugin in the
exact environment it is intended to run within. And to provide
a clean mechanism for reloading the plugin at that.

I believe SCI can facilitate REPL behavior, so I'm sure it can
be easily done.

The build script below will generate plugins into the EDN we deliver to
the client

#+begin_src clojurescript :tangle build.cljs
(ns build
  (:require ["fs" :as fs]
            ["path" :as path]
            [clojure.edn :as edn]))

(defn build []
  (let [metadata-str (fs/readFileSync "manifest.edn" "utf8")
        metadata     (edn/read-string metadata-str)
        ui-code      (fs/readFileSync "src/ui.cljs" "utf8")
        worker-code  (fs/readFileSync "src/worker.cljs" "utf8")

        payload (assoc metadata
                       :ui-form-str (str "(do\n" ui-code "\n)")
                       :worker-form-str (str "(do\n" worker-code "\n)"))

        dist-dir "dist"
        out-file (path/join dist-dir "plugin.edn")]

    (when-not (fs/existsSync dist-dir)
      (fs/mkdirSync dist-dir))

    (fs/writeFileSync out-file (pr-str payload))
    (println "Plugin packed successfully to" out-file)))

(build)
#+end_src
